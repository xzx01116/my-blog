## 二、优势与限制
1.  语法简单，没有那么多设计模式
2. 编译速度很快
3. 天生支持高并发 ：一个for循环就可以开启一万个并发程序
4. go不依赖其他库 ：go采用的是静态链接，当执行go build时，编译器会把代码、GC、并发调度器等全部打包成一个二进制文件，丢到LInux服务器上可以直接把服务跑起来
5. 所有 Exception 都用 Error 来处理，不像java可以使用try catch捕获，只是把错误看作一个普通的值

## 三、语法
`package main` ：任何go的源代码文件，第一行必须声明自己属于哪个包，只有当包名是 main 时，Go编译器才会把它当成一个可以独立运行的程序来编译

`package utils` ：编译出来的是一个供别人使用的依赖库，自己跑步起来

`import "fmt"` ：引入依赖，用到标准库里的 fmt 工具包，主要用来处理格式化的输入输出

`func main()` : 整个程序的绝对入口，如果有 `func init()`的话，会先执行init函数（比如提前初始化数据库的连接池）

`go run xxx.go` : <font color="#2DC26B">开发调试用</font>，把编译和运行合并了

`go build xxx.go` : <font color="#2DC26B">生产发布用</font>，打包成二进制可执行文件

注释 ：//   和  /*   /*
左括号不能换行，要和函数名在一行

`fmt.println() 和 fmt.print()` : 输出


## 四、声明变量

`var a int`  :  使用了“零值”机制，Go会自动赋予默认值 “0”，所以Go永远没有未初始化的变量  
`var b = 100   var c = "xzx"` : 自动类型推导

`d := 200` ： 短变量声明
ps ： 只能在函数内部使用，且左边必须有新变量，写过`d := 200`就不能再写`d := 300`了


`var x, y int` ：多变量声明
`e, f := 123, "hello"` ： 适合用来接收函数返回值

`var (`
    `globalA int`
    `globalB bool`
`)`  ： 全局变量块，通常放到 import 下面


```go
package main
import "fmt"

func main() {

    // 第一种方法，不赋值默认为0
    var a int
    var b string
    fmt.Println(a, b)

    // 第二种方法，赋值
    var c = true
    fmt.Printf("c= %t, type of c= %T\n", c, c)
    
    // 第三种方法，类型推断
    var d = 100.2
    fmt.Printf("d= %f, type of d= %T\n", d, d)
    
    // 第四种方法，省区var，使用:=
    e := "xzxx"
    fmt.Printf("e= %s, type of e= %T\n", e, e)

}

ps : Println不能解析占位符，所以要用Printf
	 一个变量声明了但是不使用也会报错
```

## 五、常量

`const length int = 10` : 常量在编译阶段就会被替代

**iota** : 用来实现枚举，它是const代码块中的行索引器，遇到const归零

```go
package main
import "fmt"

const(
    a = iota // 0
    b
    c
)

const(
    shenzhen,shanghai = iota + 1,iota + 2    // 1，2
    chengdu,beijing                          // 2，3
    hangzhou,nanjing = iota * 2,iota * 2 + 1 // 4，5
    chongqing,zhengzhou                      // 6，7
)

func main() {
    fmt.Println(a,b,c)
    fmt.Println(shenzhen,shanghai,chengdu,beijing)
    fmt.Println(hangzhou,nanjing,chongqing,zhengzhou)
}
```

```go
    return a + b
```

现在Go编译器通过go.mod文件来确定当前项目的“根命名空间”
`go mod init gostudy`  ：初始化Go Module, 给项目取个名字，以后所有和（go.mod）处于同级的文件的顶级前缀都叫 gostudy

Go中没有public和private，首字母大写代表公有，首字母小写代表私有
Go中导入包不使用也会报错

可以给包起别名，比如  `_ "gostudy/lib1"`    这可以匿名，不使用也不报错         `mylib2 "gostudy/lib2"`    // 比如github有个myysql包中的init方法可以自动注册数据库驱动，就可以使用匿名导包
   
点导入 `. xxx`：可以直接把这个包融入自己的项目中，不再需要前缀名去调用了。但是一般不使用，因为不知道用的是哪个包的方法

```go

 ├── main.go       <-- 主程序
```

## 八、指针
指针存储的是一个内存地址，在Go是只读的
`&` ：取地址符
`*` ： 解引用符

栈：高效、函数调用时压入，结束时弹出、一般存储函数内部的局部变量、参数、返回值        不需要GC介入
堆： 巨大的、全局的内存区域，可以跨函数存活，堆内存中的废弃数据必须依赖GC去扫描和清理

逃逸分析： 在 C 或 C++ 中，程序员必须手动决定内存分配在哪（用 `malloc` 分配到堆，不写就是栈），而在Go中，在编译阶段编译器会通过“逃逸分析”自动决定：如果一个变量在函数结束后，还能被函数外部引用，编译器就会把这个变量从栈提拔到堆上（即逃逸）

```go
func swap(a,b *int) {
    *a,*b = *b,*a
}

func main() {
    a,b := 10,20
    swap(&a,&b)
    fmt.Println(a,b)
}
```

## 九、defer（延迟调用）

本质 ：向 Go 编译器注册了一个“无论发生什么，在函数即将结束退出的那一刻，都必须执行的动作”
优势 ：避免忘记关闭资源，导致内存泄漏

代码解析：nil在Go就是空的意思

```go
file, err := os.Open("data.txt")  go的特色：双返回值
if err != nil {      
    return err
}
// 紧接着就写关闭逻辑，Go 会帮你“记住”它，等函数结束时再执行
defer file.Close() 

// 下面你可以放心大胆地写各种业务逻辑，随时 return，再也不用管文件关闭的事了

```


多个defer的执行顺序 ： LIFO（后进先出）----压栈

```go
func main(){
    defer fmt.Println("defer 1")
    defer fmt.Println("defer 2")
    fmt.Println("123")
    fmt.Println("456")
}
先打印defer2，再打印defer1
```
